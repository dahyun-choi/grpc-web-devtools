feat: Implement chrome.debugger API for reliable Repeat functionality

Implemented chrome.debugger API to capture raw HTTP request bodies,
enabling gRPC request repetition without proto files. Added URL-based
matching fallback to resolve ID mismatch issues between DebuggerCapture
and existing interceptor system.

## Key Changes

### chrome.debugger API Integration
- Add debugger permission to manifest.json
- Create DebuggerCapture class (src/utils/DebuggerCapture.js)
- Integrate with index.js for automatic initialization
- Capture Network.requestWillBeSent events for raw POST data

### URL-based Matching Fallback
- Implement 3-tier fallback strategy in NetworkDetails.js:
  1. requestId lookup (existing interceptor)
  2. entryId lookup (existing method)
  3. URL-based matching (new, for DebuggerCapture)
- Resolve ID mismatch between Chrome requestId and internal entryId

## Technical Details

**DebuggerCapture Flow:**
```
DevTools panel opens
  → chrome.debugger.attach(tabId)
  → Network.enable command
  → Listen for Network.requestWillBeSent events
  → Capture params.request.postData
  → Base64 encode and cache
  → Repeat button uses cached body
```

**URL Matching Logic:**
```javascript
// Match cache entries by URL comparison
for (const [cacheKey, cacheValue] of rawCache.entries()) {
  if (cacheValue.url === method ||
      cacheValue.url.includes(method) ||
      method.includes(cacheValue.url)) {
    rawRequest = cacheValue; // Found!
  }
}
```
